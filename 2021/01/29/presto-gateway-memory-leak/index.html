presto gateway 负载异常问题

发表于 2021-01-29 |

最近 presto 网关出现几次负载异常问题，导致查询延时变高，最近一次发生后，临时将问题节点从 nginx 下线，在线排查，以下记录排查过程：

排查过程

  1. 首先排查具体 jvm 哪些线程在高负载的运行，找到对应的线程堆栈可以定位到热点代码：根据 top 排序结合 jstack 信息，发现热点都是 GC 线程，怀疑是 gc 操作导致了负载变高
  1. 于是查看 gc 日志，发现 full gc 频繁，每 30s 一次，从明细来看，Old 区在 11G 左右（最大堆内存设置的 16G），并且 Old 区对象无法正常回收，怀疑存在内存泄露
  1. 使用 jmap 查看具体是哪些实例大量占用内存空间

发现占用最多的 top10 的实例都与 org.terracotta.statistics 包有关，这是 Ehcache 内嵌的统计框架使用的类，怀疑内存泄露与 Ehcache 有关

  1. 搜了下社区也有人遇到类似的问题，根据描述信息基本与我们的问题一致，目前社区给出的方法是升级 Ehcache 版本，我们已经修改灰度一台在生产运行，目前正常，继续观察。

改进措施

  • 遇到线上问题，不要急于重启，注意保留现场，必要时隔离节点，在线调试
  • 细化监控，比如 jvm gc 相关的指标，方便定位问题
  • 核心指标加上告警，比如这里的 full gc 次数等
